# 연구 배경
본 연구는 steam에서 판매 중인 게임 "**농부는 대체되었다**<sub>The Farmer Was Replaced</sub>"에 등장하는 미로 탐색 문제를 효율적으로 풀기 위해 하향식 접근을 통한 알고리즘 최적화<sub>optimization</sub>에 목적을 둔다.

# 연구 조건
### 다중 드론 기반 미로 탐색 시뮬레이션 연구 조건

#### 맵 조건
- 맵 크기 고정
    - 전체 맵과 미로의 크기는 $32 \times 32$ 공간으로 고정됨.
    - 벽의 두께는 0으로 수렴하며, $n \times n$ 미로 내 이동 가능한 공간은 총 $n^2$칸임.
- 미로 생성 규칙
    - 플레이어 위치와 무관하게 맵 전체를 대상으로 일괄 생성함.
- 초기 배치 및 사전 행동
    - 미로 생성 전 자유 이동을 통해 탐색 시작점 설정 가능. 분신 소환 및 사전 행동 수행 또한 선택적으로 가능함.
- 시간 단위
    - 미로 내 이동(1칸 전진)은 연산 복잡도와 무관하게 1틱으로 고정함.

#### 드론 인지 범위
- 인지 정보
    - 미로 생성과 동시에 드론는 자신의 현재 좌표와 목적지의 절대 좌표를 인지함.
- 시야 제한
    - 맵과 미로의 전체 크기는 인지하나, 내부 지형(벽 배치)은 직접 이동하여 탐색하기 전까지 미지 상태로 유지됨.
- 완벽한 미로
    - 모든 공간이 연결된 트리 구조로, 순환하는 루프 구간이 존재하지 않음. 따라서 좌수법이나 우수법 등 경로 탐색 알고리즘으로 전 구역 탐색이 보장됨.

#### 다중 드론 소환 및 메모리 제어
- 분신 소환
    - 탐색 도중 1틱을 소모하여 현재 위치에 새로운 드론(분신) 생성 및 탐색 로직 할당 가능.
- 독립적 메모리
    - 소환 시점의 초기 상태값(변수, 리스트 등)을 복사하여 전달함. 생성 직후부터 각 드론는 기존 드론들과 메모리를 공유하지 않는 독립적인 상태 공간을 가짐.
- 동시 소환 제한
    - 맵 내 동시 존재 가능한 최대 드론 수는 최초 드론를 포함하여 $n$명으로 제한함 ($n$은 별도 입력되는 상수).
- 소환 실패 로직
    - 제한 인원 초과 시 소환 시도는 즉시 무시되며, 기존 드론의 상태 변화나 데이터 손실은 발생하지 않음.

#### 탐색 종료
- 목표 달성 시 즉시 종료
    - 어떤 드론든 보물 위치에 도달하는 즉시 시뮬레이션이 종료된다.

## 연구 방법
1. 미로 생성
    1.엘러의 알고리즘<sub>Eller's Algorithm</sub>을 사용한 $n \times n$ 미로를 생성을 시도하였으나, 벽으로 이루어진 방(2x2 이상)과 일정한 패턴이 균일하게 생성됨을 확인함
    1. 재귀적 생성 알고리즘으로 변환
        1. 시간 복잡도 $O(N)$에서 $O(N\log{N})$으로 격상
    1. 수정후 미로 중 벽으로 이루어진 방이 생성되지 않음을 확인
    1. 클래스화하여 0과 1로 이루어진 이중 배열로 리턴
        1. 0은 벽<sub>wall</sub>
        1. 1은 길<sub>path</sub>
        1. 2는 플레이어<sub>PP : Player</sub>
        1. 3은 목적지<sub>GG : Goal</sub>
1. 하향식 접근법
- 최초 발상은 시작 드론 위치, 시작 드론 수, 알고리즘까지 비지도학습을 통한 상향식 접근을 선택하려 하였으나, 자원 및 시간의 고려로 하향식 접근법으로 전환
    1. 알고리즘 선택 및 구현
        1. 좌수법/우수법
        1. DFS<sub>Depth First Search</sub>
        1. BFS<sub>Breadth First Search</sub>
        1. D* lite
        1. 휴리스틱 병렬 DFS <sub>Heuristic Parrellel DFS</sub>
        
        - 고찰
            - 루프 없이 형성되는 본 미로 특성상 좌수법/우수법과 깊이 우선 탐색<sub>DFS</sub>은 크게 차이가 없음
            - 드론이 직접 탐사해야하는 특성상 너비 우선 탐색<sub>BFS</sub>은 Backtracking의 비용이 너무 커짐
            - D* lite 알고리즘의 경우 장애물이 있는 환경에서 길 찾기에 최적화되어있지만 미로와 같은 곳에선 역시 부담이 큼
            - DFS, BFS를 동시에 운용할 수 있는 병렬 DFS(A*와 유사함)를 선택, 휴리스틱 알고리즘을 적용하여 보다 최적화된 경로를 선택하도록 함
                - 병렬 탐색이 불가능할 경우 목적지 위치를 통해 멘해튼 거리가 적은 방향을 우선 선택하도록 함
    1. 시뮬레이션을 통한 데이터 추출
        1. 독립변수 후보
            - 초기 드론 수 - **선택**
            - 각 드론의 좌표 / 편향
            - 각 드론 응집률
            - 목적지와 실제 틱 수 
            - 맨해튼 휴리스틱 거리
        1. 종속변수 후보
            - 사용된 틱 수 - **선택**
            - $\text{최적화 된 경로} - \text{사용된 틱 수}$ - **선택**
    1. 데이터 전처리
        - DataFrame의 Success 칼럼 중 False를 제거(이상치 처리)
            - 1회차 실험 중 데이터의 약 0.1%가 False로 나타나어 실패원인을 분석
                - 재귀적 미로 생성 알고리즘에서 함수 매개변수 전달에 리스트를 넣는 과정에서 얕은 복사를 사용하지 않은 것을 확인
                - 이로 인해 목적지로 향하는 길이 단절된 미로가 생성되는 버그를 확인하고 `.copy()`를 사용하여 수정함.
            - 2회차에서는 False인 행이 총 0건 발견되어 데이터를 전부 사용함
        - 일부 사용된 틱 수가 비정상적으로 높은 값을 확인
            - 이상치를 삭제하는 것이 아닌 유효한 데이터로 판단함
            - A* 알고리즘으로 최적화 된 경로를 구하고, 최적화된 사용된 틱을 계산 하여 `Optimal_Tick` 컬럼으로 저장
            - 최적화된 틱과 사용한 틱의 차로 낭비된 틱을 새로운 칼럼으로 추가
            $$\text{Waste Tick} =  \text{Used Tick}- \text{Optimal Tick}$$
    1. 데이터 분석
        - 모집단이 세종류 이상이므로 ANOVA(일원분산분석)을 사용하여 각 모집단의 분산성을 판단

    1. 데이터 검증
        - OLS(최소제곱법) 검증을 통한 데이터 검증
            - $R^2$값을 통한 모델 설명력 검증
            - Q-Q plot을 통한 데이터 정규성 검증
        
## 연구 결과
1. 시뮬레이션 결과
    - 약 129분을 사용하여 시뮬레이션을 돌린 결과 $920,820 \times 12$의 데이터를 획득
        - DataFrame columns setting
            - Seed: 맵 생성시 고정되는 난수 시드
            - Spawn_Strategy: 초기에 배치하는 드론 수
            - Initial_Drones_XY: 초기에 배치하는 드론의 좌표 리스트 `[(x,y)]`
            - Goal_X: 목적지의 x좌표
            - Goal_Y: 목적지의 y좌표
            - Optimal_Tick: 사전 계산된 이상적인 최소 틱
            - Used_Tick: 탐색 시작으로부터 목적지를 발견하기까지의 소요된 틱
            - Map_Coverage_%: 드론이 탐색한 맵의 백분위
            - Spawn_Fail_Count: 미로 분기접에서 드론 병렬 소환에 실패하여 직접 탐색에 참가하는 횟수
            - Backtrack_Count: 드론이 드론을 소환하지 못하여 직접 탐색할 때 막힌 길을 마주하여 되돌아간 횟수
            - Total_Spawned: 한 루프에서 사용된 드론 수
            - Success : 탐색 성공 여부
